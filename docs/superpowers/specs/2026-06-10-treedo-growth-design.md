# treedo growth stages — design

2026-06-10. Approved in-session.

## Goal

The treedo tree's size reflects the todo count: a one-leaf seedling at 1
todo, growing through hand-drawn stages to the current full tree at 100
todos, and shrinking back down as todos are removed. An empty list shows
a seed mound. Stage changes animate (~0.8s scale tween); within a band
the tree rests at its hand-drawn, pixel-perfect size.

## Decisions (user-confirmed)

- Full size at **100 todos**; seedling with exactly one leaf at 1 todo.
- **Hand-drawn stages** (distinct hand-tuned templates per stage), not
  continuous morphing — maximal art control; the branch-layout change at
  a stage swap is an accepted "pop", smoothed by the size tween.
- **Animated transitions** between stages; instant (no tween) at startup.
- Empty list (0 todos) shows a **seed mound** where the trunk stood.

## Stage table

| Stage | Count band | Trunk (h×w) | Branches | Leaf slots |
|---|---|---|---|---|
| S0 seed mound | 0 | — | — | 0 |
| S1 seedling | 1–4 | 36×4 stem | 0 | 2 (explicit, at stem tip) |
| S2 sprout | 5–19 | 110×10 | 2 | 6 |
| S3 young | 20–49 | 200×16 | 4 | 14 |
| S4 mature | 50–99 | 270×22 | 6 | 24 |
| S5 full | 100+ | 320×26 | 8 | 34 — today's tree unchanged |

Geometry values are starting points, tuned against real screenshots
during implementation. Each stage template carries: trunk `{h w lit-w}`,
branch list (same `[ax ay tx ty nslots]` format as today), branch base
thickness, root strokes, and knot positions. Smaller stages have fewer,
thinner roots and no knots; S5 reproduces the current tree byte-for-byte
at rest.

## Mechanics

- **Stage selection**: pure function of `(length items)` → stage index,
  evaluated each frame in the `tree-of-life` canvas from the existing
  `:items` subscription. No dataflow/handler/view changes.
- **Slots**: per-stage, generated by the existing `branch-slots` +
  ring-interleave logic run against the stage's branch template
  (seedling slots are explicit). Items keep the `mod nslots` wrap;
  falling leaves keep `entry.slot mod nslots`. Leaves stack on slots
  when a band holds more todos than slots, as today.
- **Transition tween**: closure-local `var`s in the canvas section hold
  the shown stage and current scale. On a band crossing, switch to the
  new template at scale `old-visual-height / new-stage-height` (so the
  visual height is continuous), then ease scale → 1.0 over ~0.8s
  (ease-out, dt from `redin.now`; snap when within 1%). All template
  offsets multiply by the scale, anchored at the trunk base `(bx, by)` —
  slots scale too, so leaves ride the branches. Rapid crossings retarget
  from the current scale. First frame initializes settled (seeding 100
  shows the full tree with no animation). S0's nominal height is ~10px
  so the seedling visibly shrinks into the mound.

## Out of scope

Dataflow, handlers, view, journal card, forest backdrop, vine/grip
canvases, `TREEDO_ITEMS` seeding — all unchanged. The feature lives
entirely in the tree-geometry / `tree-of-life` sections of
`examples/treedo.fnl`.

## Verification

Canvas-only change (no state to assert), so verification is visual:
screenshots at `TREEDO_ITEMS` = 0, 1, 5, 20, 50, 100 for each rest
stage, plus a mid-tween screenshot after dispatching `test/add` /
`test/remove` across a band threshold. Examples have no automated test
convention in this repo; the runtime test suite must stay green
(unaffected).
